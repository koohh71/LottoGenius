# FTP 동기화 프로그램 요구사항 정의서 (Draft)

## 1. 프로젝트 개요
- **프로젝트명:** FtpSyncApp (가칭)
- **목표:** 사용자 인증 절차를 거친 후, 지정된 로컬 폴더의 상태를 원격 FTP 서버의 특정 폴더와 100% 일치(Mirroring)시키는 데스크톱 애플리케이션 개발.
- **타겟 OS:** Windows

## 2. 주요 기능 요구사항

### 2.1 사용자 인증 (로그인)
- 애플리케이션 실행 시 로그인 화면 출력.
- 사용자 ID와 비밀번호를 입력받아 인증 수행.
- *질문: 인증은 FTP 서버 계정으로 직접 하는 것인가요, 아니면 별도의 관리자 서버가 있나요? (현재는 FTP 접속 정보를 이용한 인증으로 가정)*

### 2.2 동기화 (Synchronization)
- **방향:** Remote(원격) → Local(로컬) (단방향 미러링)
- **동작 방식:**
  1. 원격지 파일/폴더 리스트 스캔.
  2. 로컬 대상 폴더 리스트 스캔.
  3. **다운로드:** 원격지에는 존재하지만 로컬에 없거나, 내용이 변경된 파일 다운로드.
  4. **삭제:** 로컬에는 존재하지만 원격지에는 없는 파일/폴더 삭제 (원격과 100% 일치 보장).
- **무결성 검사:** 파일 크기 및 수정 시간을 기준으로 변경 여부 판단 (필요 시 해시 비교 도입).

### 2.3 설정 및 UI
- 동기화 진행 상황 표시 (Progress Bar, 로그).
- 로컬 저장소 경로 설정 (고정 경로 또는 설정 파일/UI에서 지정 가능하도록).

## 3. 기술 스택 제안 (선택 필요)
빠른 개발과 윈도우 호환성을 위해 아래 스택을 제안합니다.

- **언어:** Python 3.10+
- **GUI 프레임워크:** PyQt6 또는 PySide6 (모던한 UI 제공 용이)
- **통신:** Python 내장 `ftplib` 또는 `SFTP` 필요 시 `paramiko`
- **배포:** PyInstaller (단일 EXE 파일로 배포)

- 동기화가 자동으로 주기적으로 수행되어야 하나요, 아니면 로그인 직후 1회만 수행하면 되나요?
